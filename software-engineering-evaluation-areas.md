# Software Engineering Evaluation Areas

20 areas across 5 categories for evaluating software engineering capabilities.

---

## Creation

| Area | Description |
|------|-------------|
| **Greenfield** | Starting from scratch with no existing codebase |
| **Prototyping/Spike** | Quick throwaway code to test feasibility |
| **Architecture** | High-level system design, component boundaries, trade-off decisions |
| **API Design** | Designing interfaces, contracts, and public-facing APIs |
| **Data Modelling** | Schema design, data structures, database design |

---

## Evolution

| Area | Description |
|------|-------------|
| **Maintenance** | Keeping existing systems running |
| **Refactoring** | Restructuring abstractions/boundaries (class level and up) without changing behaviour |
| **Rewriting** | Replacing a function/method with new implementation in the same language |
| **Porting** | Moving code to new platforms/languages while preserving functionality |
| **Code Migration** | Upgrading dependencies, framework versions, or language versions |

---

## Quality

| Area | Description |
|------|-------------|
| **Debugging** | Tracing issues to identify *why* something is failing |
| **Bug Fixing** | Implementing the fix once the root cause is known |
| **Testing** | Writing unit tests, integration tests, or fixing test suites |
| **Code Review** | Evaluating others' code for quality, bugs, and standards |
| **Performance Optimisation** | Improving speed, memory usage, or efficiency |
| **Security** | Identifying vulnerabilities, implementing secure patterns |
| **Concurrency** | Handling parallelism, async patterns, race conditions |

---

## Knowledge

| Area | Description |
|------|-------------|
| **Documentation** | Writing/updating technical docs, comments, READMEs |
| **Legacy Code Comprehension** | Understanding unfamiliar or old codebases |

---

## Operations

| Area | Description |
|------|-------------|
| **Infrastructure** | Managing deployment and infrastructure as code |
